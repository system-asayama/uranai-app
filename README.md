# uranai-app（三国志占い）

生年月日と時刻から **四柱推命の命式（年柱・月柱・日柱・時柱）** を割り出し、
日柱の六十干支（60パターン）から **あなたに宿る三国志の英雄** を導く占い
Web アプリです。性格鑑定・五行バランス、二人の相性占いもできます。
Flask + SQLAlchemy で実装。簡易的なログイン認証 / ユーザー管理も備えています。

## 占い機能

- 生年月日（＋任意で時刻）を入力すると、命式と **三国志キャラ** を表示
  - 占いページ: `/uranai`（ログイン不要）
  - 三国志キャラ六十干支の早見表: `/uranai/list`（ログイン不要）
  - 相性占い: `/aisho`（**ログイン必須**）
- 表示内容
  - **三国志キャラ**：日柱の干支に対応する英雄（勢力・肩書き・割り当て理由）
  - **命式表**：年柱・月柱・日柱・時柱の天干／地支（五行で色分け）
  - **性格鑑定**：六十干支（甲子〜癸亥）60 パターンの性格・仕事・恋愛・ラッキー
  - **五行バランス**：命式 8 文字（時刻不明なら 6 文字）の五行の数
- 出生時刻は任意。不明な場合は時柱を省略します。

### 三国志キャラの割り当てについて

三国志の人物は、生年（年）すら不明な者も多く、日柱の算出に必要な
「年・月・日」までそろった正確な生年月日はほぼ誰も史料に残っていません
（巷で見かける誕生日の多くは旧暦かつ後世の創作で諸説あり）。
そのため本占いでは、各人物を **史実・性格・五行のイメージ** から
六十干支に割り当てています（`sangokushi.py`）。日付レベルの生年月日が確実に
分かる人物がいれば、その干支へ差し替えられる構造にしてあります。
60 キャラは魏・蜀・呉・群雄からバランス良く選定しています。

### 四柱の計算方法

時刻はすべて日本標準時 (JST = UT+9) で扱います。

- **年柱**：元日ではなく **立春** で切り替え。立春前は前年の干支。
  年干支 = `(立春で区切った年 − 4) % 60`（甲子 = 0）。
- **月柱**：**節入り（節気）** で切り替え。太陽の視黄経を天文計算
  （Meeus 簡易式＋ΔT 補正）で求め、立春(黄経315°)を寅月の起点として
  30°ごとに月の地支を決定。月の天干は五虎遁（年干から導出）。
- **日柱**：ユリウス通日 (JDN) から `(JDN + 49) % 60`。
  **日付の変わり目は 23 時**（子の刻）。23 時以降は翌日の干支。
- **時柱**：2 時間ごとの十二支（23–1 時＝子）。天干は五鼠遁（日干から導出）。

検証例：

| 生年月日（時刻） | 年柱 | 月柱 | 日柱 | 時柱 |
| --- | --- | --- | --- | --- |
| 2000-01-01 02:30 | 己卯 | 丙子 | 戊午 | 癸丑 |
| 1987-06-10 | – | – | 庚寅 | – |

> ※年柱・月柱の境界（立春・節入り）は天文簡易式のため、節入り前後
> 数分以内に生まれた場合はプロの万年暦と結果が分かれることがあります。
> 三国志キャラと性格鑑定は四柱の中心となる日柱（六十干支）にもとづきます。
> 蔵干・通変星までは扱いません。なお日本がグレゴリオ暦を採用した
> 1873 年以降に対応。

### 相性占いの判定方法

二人の日柱を中心に、日干の五行関係（相生・相剋・比和）、日支の合・冲・害・
三合、干合を総合してスコア（0〜100）と相性ランクを算出します。

## ログイン / ユーザー管理機能

- ユーザー登録 / ログイン / ログアウト（セッションベース認証）
- パスワードはハッシュ化して保存
- ロールによるアクセス制御（`admin` / `user`）
  - 利用者ログイン: `/login`
  - 管理者ログイン: `/admin/login`
- 管理者向けユーザー管理画面（一覧・作成・ロール変更・削除）
  - ※最後の管理者は削除・降格できない安全装置付き

## 初期管理者アカウント

起動時に管理者アカウントが自動作成されます（既存なら何もしません）。

| 項目 | デフォルト | 環境変数 |
| --- | --- | --- |
| ユーザー名 | `admin` | `ADMIN_USERNAME` |
| パスワード | `admin123` | `ADMIN_PASSWORD` |

本番では必ず `ADMIN_PASSWORD` と `SECRET_KEY` を変更してください。

## 起動方法

### Docker Compose（Flask + PostgreSQL）

```bash
docker compose up --build
```

http://localhost:8000 にアクセスします。

### ローカル単体実行（SQLite にフォールバック）

`DATABASE_URL` が未設定の場合は SQLite (`app.db`) を使います。

```bash
pip install -r requirements.txt
python app.py
```

## 環境変数

| 変数 | 説明 |
| --- | --- |
| `DATABASE_URL` | DB 接続先。未設定なら SQLite を使用 |
| `SECRET_KEY` | セッション署名鍵。本番では必ず変更 |
| `ADMIN_USERNAME` | 初期管理者のユーザー名 |
| `ADMIN_PASSWORD` | 初期管理者のパスワード |
